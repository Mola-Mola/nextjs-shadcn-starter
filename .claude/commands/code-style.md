# Claude 명령어: Code Style

프로젝트의 전체적인 코드 스타일과 현재 파일을 비교하여 일관성을 확인하고 개선안을 제안합니다.

## 사용법

```
/code-style
```

또는 특정 파일 지정:

```
/code-style path/to/file.tsx
```

## 분석 프로세스

1. **현재 파일 분석**
   - 대상 파일의 코드 스타일 파악
   - 네이밍 컨벤션, 구조, 포맷 확인

2. **프로젝트 패턴 조사**
   - 유사한 파일들 검색 (컴포넌트, 훅, API 등)
   - 프로젝트 전체의 일관된 패턴 파악
   - 설정 파일 확인 (eslint, prettier, tsconfig 등)

3. **차이점 식별**
   - 네이밍 규칙 (camelCase, PascalCase, kebab-case)
   - 파일 구조 및 import 순서
   - 들여쓰기 및 포맷팅
   - 타입 정의 방식
   - 주석 스타일
   - 에러 핸들링 패턴

4. **개선안 제안**
   - 발견된 차이점 목록화
   - 프로젝트 표준에 맞는 코드 제안
   - 우선순위별 개선사항 정리

## 검사 항목

### 코드 구조
- 파일 및 폴더 네이밍
- Import 문 정렬 및 그룹핑
- Export 방식 (named vs default)
- 컴포넌트 구조 (함수형 vs 클래스)

### 네이밍 컨벤션
- 변수명 (camelCase)
- 함수명 (camelCase)
- 컴포넌트명 (PascalCase)
- 상수명 (UPPER_SNAKE_CASE)
- 타입/인터페이스명 (PascalCase)

### 타입스크립트
- any 타입 사용 여부
- 타입 정의 vs 인터페이스
- 제네릭 사용 패턴
- 타입 import 방식

### 스타일링
- CSS 방식 (Tailwind, CSS Modules, Styled Components 등)
- 클래스명 규칙
- 반응형 디자인 패턴

### React 패턴
- Hooks 사용 패턴
- Props 타입 정의
- 상태 관리 방식
- 이벤트 핸들러 네이밍

### 에러 핸들링
- try-catch 패턴
- 에러 바운더리 사용
- 에러 메시지 처리

## 출력 형식

```markdown
## 코드 스타일 분석 결과

### ✅ 일치하는 패턴
- [항목별 프로젝트 표준과 일치하는 부분]

### ⚠️ 개선이 필요한 부분
1. **[카테고리]**: [문제점]
   - 현재: [현재 코드 예시]
   - 제안: [개선된 코드 예시]
   - 이유: [왜 이렇게 해야 하는지]

### 📊 우선순위
- 🔴 높음: [즉시 수정 필요]
- 🟡 중간: [개선 권장]
- 🟢 낮음: [선택적 개선]
```

## 참고사항

- 프로젝트의 eslint, prettier 설정 우선 적용
- CLAUDE.md의 코딩 스타일 가이드 준수
- 일관성을 해치지 않는 범위 내에서 제안
- 설명과 함께 실제 적용 가능한 코드 제공
